# FileCheckSkill — Приём и анализ макетов РПК «ПинГвин»

## Главное правило

Клиент прислал **фото, картинку или файл** → **обязательно проанализируй вложение** (vision для изображений, read для текстовых/PDF если доступно).

- Дай **первичную техоценку** в том же ответе — не «передадим специалисту и вернёмся».
- Финальное «годится / не годится к печати» — только после менеджера/технолога; твоя оценка — **предварительная**.

---

## Что анализировать (изображения / фото макета)

Смотри на вложение и опиши по пунктам, что **видишь**:

| Пункт | Что сказать клиенту |
|--------|---------------------|
| Содержание | Что на макете (логотип, текст, визитка, вывеска, баннер…) |
| Формат / пропорции | Ориентир размера (90×50, A4, квадрат…) — если видно или клиент указал |
| Текст | Читается ли; опечатки на вид; мелкий шрифт |
| Цвет | RGB/«как на экране» vs печать CMYK — предупреди о возможном сдвиге |
| Качество | Размытие, пиксели, скрин с телефона, водяные знаки |
| Вылеты / поля | Есть ли запас под обрез (полиграфия — обычно +3–5 мм) |
| Формат файла | JPG/PNG фото макета ≠ готовый PDF/AI — объясни, если нужен исходник |

**Не выдумывай dpi и точные мм**, если их нельзя определить — пиши «по картинке не видно, нужен исходник PDF/AI».

---

## Что анализировать (файлы)

| Тип | Действие |
|-----|----------|
| PDF, AI, CDR, EPS, TIFF | Прочитать/открыть если инструмент доступен; иначе — формат + размер + «открыли, на предпросмотр» |
| PNG, JPEG | Vision + проверка: похоже на финальный макет или черновик/скрин |
| DOC, DOCX, XLS | Извлечь суть (ТЗ, контакты, таблица тиражей) |
| Неизвестный / битый | Попросить PDF Press Quality или AI/CDR |

## Принимаемые форматы

| Формат | Предпочтительный | Примечание |
|--------|-----------------|------------|
| PDF Press Quality | Да | Готов к печати |
| AI (Adobe Illustrator) | Да | Векторный |
| CDR (CorelDRAW) | Да | Векторный |
| EPS | Да | Векторный |
| TIFF | Да | Без сжатия |
| PNG | Условно | При разрешении ≥ 300 dpi |
| JPEG | Условно | При разрешении ≥ 300 dpi |
| Фото в Telegram | Да | Предварительный просмотр; для печати нужен исходник |

## Когда сразу handoff (file_review)

- Герб, печать организации, юр. макеты
- Клиент настаивает «это финальный макет», а видны явные дефекты — мягко + handoff
- Нестандартная продукция, которую нельзя оценить по фото

---

## Алгоритм

```
1. Получить вложение из Telegram/email (изображение попадает в контекст — проанализируй vision)
2. Провести первичный анализ (таблицы выше)
3. Сохранить: uploads/[YYYY-MM]/[session_id]_[filename] (если есть путь к файлу)
4. Ответ клиенту — шаблон выше, в том же сообщении
5. notify_managers.sh — кратко: клиент, тип файла, вывод предпросмотра, заявка
6. Привязать к обращению в СБИС (ДопПоля.МакетыФайлы) после create_appeal
7. Продолжить сбор ТЗ / расчёт
```

| Услуга | Разрешение | Цвет | Примечание |

| Полиграфия | 300 dpi | CMYK | Шрифты в кривых, +5 мм под обрезку |
