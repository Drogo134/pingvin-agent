# MemorySkill — Долгосрочная память агента

## Структура

```
memory/
  global_rules.md          ← общие правила стиля (≤25)
  clients/
    telegram_123456789.md  ← правила для конкретного клиента (≤15)
  sessions/
    20260528_123456789.txt ← сохранённые саммари диалогов
```

## Когда читать память (обязательно)

**В НАЧАЛЕ каждого Telegram-диалога** выполни два действия:

1. Прочитай `memory/global_rules.md` — применяй все пункты к стилю ответов.
2. Прочитай `memory/clients/telegram_{CHAT_ID}.md` (где `CHAT_ID` — числовой ID отправителя).
   Если файла нет — это первый диалог с клиентом, ничего страшного.

> Правила из памяти **дополняют**, не заменяют AGENTS.md и VOICE.md.
> При противоречии — AGENTS.md приоритетнее.

## Когда обновлять память (обязательно)

Обновляй ПОСЛЕ каждого из событий:

| Событие | Действие |
|---------|----------|
| `create_appeal` выполнен успешно | Сохранить саммари + вызвать `update_memory.py` |
| Handoff (передача менеджеру) | То же самое |
| Клиент написал `/new` или завершил диалог | То же самое |

## Как обновлять (последовательность exec-вызовов)

### Шаг 1 — Сохранить саммари диалога в файл

```python
# Напиши краткое резюме диалога (3-8 предложений):
# - что хотел клиент
# - как он общался (стиль, скорость, что нравилось/не нравилось)
# - результат (создано обращение / handoff / ушёл без заказа)
# Сохрани в файл:
python -c "
import sys, pathlib, datetime
chat_id = 'telegram:XXXXXX'
summary = '''ЗДЕСЬ ТЕКСТ САММАРИ'''
ts = datetime.datetime.now().strftime('%Y%m%d_%H%M%S')
safe = chat_id.replace(':', '_')
path = pathlib.Path('memory/sessions') / f'{ts}_{safe}.txt'
path.parent.mkdir(exist_ok=True)
path.write_text(summary, encoding='utf-8')
print(f'Saved: {path}')
"
```

### Шаг 2 — Обновить память

```bash
python scripts/update_memory.py \
  --chat-id telegram:XXXXXX \
  --summary-file memory/sessions/<имя_файла_из_шага_1>.txt
```

Скрипт сам вызовет LLM, извлечёт правила и обновит оба файла памяти.

## Лимиты (чтобы память не росла бесконечно)

| Файл | Максимум |
|------|----------|
| `global_rules.md` | 25 правил |
| `clients/telegram_*.md` | 15 правил на клиента |

При превышении самые старые правила вытесняются новыми автоматически.

## Разделение памяти (критично)

**Никогда не смешивай**: клиентские правила пишутся ТОЛЬКО в файл этого клиента.
Файл именуется по Telegram chat_id — уникально для каждого пользователя.

Пример:
- Клиент A (`telegram:111`): `memory/clients/telegram_111.md`
- Клиент Б (`telegram:222`): `memory/clients/telegram_222.md`

Агент читает только тот файл, чей chat_id совпадает с текущим отправителем.
