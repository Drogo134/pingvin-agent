# EmailIntakeSkill — Приём заявок по email

## Cron-расписание

```
*/5 * * * *  → проверять INBOX каждые 5 минут
```

## IMAP конфигурация

```
EMAIL_IMAP_HOST     = ${EMAIL_IMAP_HOST}     # imap.yandex.ru
EMAIL_IMAP_PORT     = ${EMAIL_IMAP_PORT}     # 993
EMAIL_IMAP_USER     = ${EMAIL_IMAP_USER}
EMAIL_IMAP_PASSWORD = ${EMAIL_IMAP_PASSWORD}
EMAIL_IMAP_TLS      = true
EMAIL_MAILBOX       = INBOX

SMTP_HOST     = ${EMAIL_SMTP_HOST}           # smtp.yandex.ru
SMTP_PORT     = ${EMAIL_SMTP_PORT}           # 465
```

## Алгоритм обработки

### 1. Получить непрочитанные письма

```
IMAP SEARCH UNSEEN → список message_id
Для каждого нового:
  - Извлечь: From, Subject, Body, Attachments, Date
  - Пометить как прочитанное
```

### 2. Классификация письма

| Тип | Признаки | Действие |
|-----|----------|---------|
| Заявка с сайта | Тема содержит "заявка", "форма", "обращение" или есть поля Имя/Телефон/Сообщение | Обработать как лид |
| Прямое обращение | Свободный текст от клиента | Обработать как лид |
| Ответ на email агента | In-Reply-To совпадает с отправленным | Добавить в существующий диалог |
| Спам / служебное | Автоматические уведомления, рассылки | Пропустить |

### 3. Извлечение данных из письма

```
Имя клиента    → из подписи или поля "Имя"
Email          → из заголовка From
Телефон        → из тела письма (regex)
Суть запроса   → тело письма
Тип услуги     → классифицировать по ключевым словам
Вложения       → макеты (передать в FileCheckSkill)
```

### 4. Автоответ клиенту (≤ 5 минут в рабочее время)

**В рабочее время (9:00–18:00 МСК):**
```
Тема: Re: [оригинальная тема]

Здравствуйте[, Имя]!

Получили ваше обращение. [Краткое резюме понятого: «Вы интересуетесь [услуга].»]

Чтобы подготовить расчёт, уточните, пожалуйста:
[1-3 уточняющих вопроса по матрице параметров]

Если удобнее общаться в Telegram: @[bot_username]

С уважением,
Команда РПК «ПинГвин», Домодедово
[телефон если есть в конфиге]
```

**В нерабочее время:**
```
Здравствуйте[, Имя]!

Ваше обращение получено. Мы работаем с 9:00 до 18:00 МСК.
Менеджер свяжется с вами в рабочие часы.

Чтобы ускорить процесс, можете сразу уточнить:
[1-2 ключевых вопроса]

РПК «ПинГвин»
```

### 5. Параллельная обработка

Одновременно с автоответом:
- Запустить SalesIntakeSkill для разбора запроса
- Создать лид в СБИС (CRMSkill)
- Если есть вложения → FileCheckSkill

### 6. Создание сессии

```json
{
  "source": "email",
  "session_id": "email_[message_id_hash]",
  "client_email": "[from_address]",
  "client_name": "[extracted_name]",
  "initial_message": "[email_body]",
  "attachments": ["[filename]"]
}
```

## Правила

- Отвечать на каждое письмо максимум через 5 мин в рабочее время
- Внутренние письма (от корпоративного домена) → не обрабатывать как заявки
- Уже отвеченные письма → не дублировать (проверять по message_id)
- При письме с макетом → FileCheckSkill перед ответом
