# FollowUpSkill — Автоматические касания

## Расписание cron

```
*/30 * * * *  → проверять очередь follow-up каждые 30 минут
```

## Условия для отправки касания

Касание ОТПРАВЛЯЕТСЯ если:
- С последнего сообщения клиента прошло 24 ч (касание 1) или 48 ч (касание 2)
- Диалог не завершён сделкой (статус ≠ `won`)
- Нет активного handoff (статус ≠ `waiting_manager`)

Касание НЕ отправляется если:
- Сделка создана (статус `won`)
- Клиент явно отказался (статус `lost`)
- Клиент ответил после предыдущего касания
- Уже было 2 касания без ответа → статус `no_response`

## Шаблоны сообщений

### Follow-up 1 (через 24 часа)

```
Здравствуйте! Возвращаюсь по вашей заявке.
Если задача ещё актуальна — могу продолжить расчёт
или передать параметры менеджеру.
```

### Follow-up 2 (через 48 часов)

```
Напомню про вашу заявку.
Можем быстро уточнить недостающие параметры
и подготовить предварительный расчёт.
```

## Алгоритм

```
1. Получить диалоги со статусом "pending_followup"
2. Для каждого:
   a. Прошло ли нужное время?
   b. Нет ли активного handoff?
   c. Нет ли закрытой сделки?
3. Если ДА → отправить шаблон через канал диалога
4. Обновить счётчик касаний (+1)
5. После 2 касаний без ответа → статус "no_response"
   → уведомить менеджера: "Клиент не ответил на 2 касания"
```

## Обновление лида в СБИС

После отправки касания:
```json
{
  "action": "update_lead",
  "data": {
    "ДопПоля": {
      "КасанийОтправлено": 1,
      "ПоследнееКасание": "[ISO8601]",
      "СледующееКасание": "[ISO8601 +24h или +48h]"
    }
  }
}
```
