# HandoffSkill — Передача менеджеру РПК «ПинГвин»

## Триггеры (любой из)

| # | Триггер | Приоритет |
|---|---------|-----------|
| 1 | Клиент явно просит: «позовите менеджера», «хочу с человеком», «соедините с оператором» | Немедленно |
| 2 | Нет тарифа в pricing-config.json | Немедленно |
| 3 | Confidence < 0.75 | Немедленно |
| 4 | Клиент квалифицирован и готов к сделке | Немедленно |
| 5 | Гербовая печать или специальные юридические требования | Немедленно |
| 6 | Запрос о скидках / специальных условиях | Немедленно |
| 7 | Нестандартный формат, нестандартный материал | Немедленно |

## Алгоритм выполнения (порядок строгий)

### 1. Сообщить клиенту

```
Чтобы ответить без ошибок, подключаем специалиста.
Мы уже передали менеджеру параметры вашей заявки и файлы,
чтобы не пришлось повторять всё заново.

[В рабочее время 9:00–18:00 МСК]:
Менеджер свяжется с вами в течение минуты.

[В нерабочее время]:
Менеджер свяжется с вами в рабочие часы с 9:00 до 18:00 МСК.
Чтобы не терять время, я уже зафиксировал все параметры вашей заявки.
```

### 2. Сформировать резюме и отправить менеджерам в Telegram

**Получатели:** все ID из `MANAGER_TELEGRAM_CHAT_IDS` (через запятую), например: `1280280963,987654321`
**Один ID (устаревшее):** `MANAGER_TELEGRAM_CHAT_ID` — используется, если список не задан
**Максимальное время:** ≤ 1 минута после триггера

**Отправка (Windows — Python):**
```bash
python scripts/notify_managers.py "[текст резюме]"
```

**Отправка (Docker / Linux — bash):**
```bash
bash scripts/notify_managers.sh "[текст резюме]"
```

```
🔔 Новый лид — Передача менеджеру

👤 Клиент: [имя/@username/email]
📱 Канал: [Telegram/WhatsApp/Email/Сайт]
🕐 Время: [дата и время]

📋 Тип заказа: [тип услуги]

📐 Параметры:
[каждый параметр с новой строки]

💰 Стоимость (Pricing Engine): [сумма или «расчёт не сделан»]
📅 Сроки: [если подтверждены, иначе —]

🔁 Причина передачи:
[confidence_low / no_tariff / client_request / qualified_lead / nonstandard / discount_request]

📝 Резюме диалога:
[3-5 предложений]

🔗 Лид в СБИС: [ссылка или «создаётся»]
```

### 3. Обращение в Saby (CRMSkill)

**Перед handoff**, если в сессии ещё нет `appeal_id` и в переписке есть минимум ТЗ (услуга + параметры + контакт):

1. `create_appeal` из истории чата (см. `skills/crm-sbis/SKILL.md`).
2. `submit_appeal` с тем же `appeal_id`.
3. `update_appeal` со статусом `waiting_manager` и резюме handoff.

Если `appeal_id` уже есть — только `update_appeal`. Не дублировать обращения.

### 4. Заблокировать самостоятельные ответы по теме

После handoff агент:
- **НЕ** даёт новых ответов по заблокированной теме (цены, сроки, условия)
- Отвечает на вопросы по другим темам нормально
- При настойчивости: «Менеджер уже работает с вашим запросом, скоро свяжется»

## Коды причин handoff

| Код | Описание |
|-----|----------|
| `confidence_low` | Уверенность агента ниже порога 0.75 |
| `no_tariff` | Нет тарифа в pricing-config |
| `client_request` | Клиент явно попросил менеджера |
| `qualified_lead` | Клиент прошёл квалификацию, готов к сделке |
| `no_kb_data` | Нет данных в базе знаний |
| `nonstandard_order` | Нестандартный заказ |
| `file_review` | Требуется проверка макета специалистом |
| `legal_seal` | Гербовая/юридическая печать |
| `discount_request` | Запрос скидки/спецусловий |

## Когда уходит Telegram-уведомление менеджерам

| Событие | Когда | Тип |
|---------|--------|-----|
| **Новый лид** | Параметры заказа собраны + цена озвучена | Информационное — диалог продолжается |
| **Handoff** | Сработал любой handoff-триггер | Полный — агент ждёт менеджера |
| **Напоминание** | Handoff >2 ч, менеджер не ответил, рабочее время 9–18 | Повторное |
| **Утренняя сводка** | Cron ~9:00 МСК, пн–пт | Авто |
| **Итоги дня** | Cron ~18:00 МСК, пн–пт | Авто |

**Шаблон для нового лида (не handoff):**
```
Новый лид — RПК ПинГвин

Клиент: [имя/@username]
Источник: Telegram
Услуга: [тип]
Параметры: [кол-во, формат, дизайн]
Предв. стоимость: ~X ₽
```

**Handoff-триггеры (полный handoff + резюме):**
1. Клиент просит менеджера / человека
2. Нет тарифа в pricing-config
3. Confidence < 0.75
4. Клиент квалифицирован, готов к сделке
5. Гербовая печать / юр. требования
6. Запрос скидки или спецусловий
7. Нестандартный заказ / материал

---

## Нерабочее время

Если handoff вне 9:00–18:00 МСК:
1. Уведомление менеджеру **всё равно отправляется** (прочитают утром)
2. Клиенту: «Менеджер свяжется в рабочие часы с 9:00 МСК»
3. Агент продолжает принимать дополнительные данные
4. Все параметры сохраняются в лид в СБИС

## Повторное уведомление

Если менеджер не ответил >2 часов в рабочее время:
→ Повтор через `notify_managers.sh` всем ID: «⚠️ Напоминание: клиент ожидает ответа [N часов]»
