# Куда попадают обращения в Saby (общая очередь vs менеджер)

**Подробное руководство:** `scripts/SBIS-RECIPIENTS-GUIDE.md` — кого назначить, три схемы (очередь / менеджер / контракт), обходные пути.

Агент создаёт документ регламента **«Обращение»** — это тот же тип, что в UI:  
[Контакт-центр → Обращения](https://online.sbis.ru/page/support-service).

## Два шага (важно)

| Шаг | Что делает | Без этого |
|-----|------------|-----------|
| `create_appeal` | Создаёт черновик | В списке Контакт-центра **пусто** |
| `submit_appeal` | Отправляет в документооборот | Остаётся «Ожидается отправка» |

## Вариант A — **общая очередь** (рекомендуется)

Обращения видят **все менеджеры** подразделения (как «общие» входящие), без привязки к одному человеку.

В `.env`:

```env
SBIS_RECIPIENT_MODE=auto
SBIS_DEPARTMENT_NAME=Контакт-центр
# или точный код/UUID подразделения из Saby:
# SBIS_DEPARTMENT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
SBIS_AUTO_SUBMIT=true
```

## Как узнать подразделение (без UI)

**Автоматически (рекомендуется):**

```powershell
python scripts\sbis_discover.py
```

См. также `scripts/SBIS-SETUP.md` — там про «ИИ Агент» vs получателя и права `ai_agent`.

**Вручную (если нужно):**

1. В Saby вручную создайте обращение и назначьте **подразделение** (общую очередь), как вы хотите видеть заявки от бота.
2. Откройте документ → скопируйте `guid` из URL.
3. Выполните:

```powershell
python scripts\sbis_show_recipient.py <guid>
```

Смотрите в ответе `document.Подразделение` или поля этапа «Отправка».

## Вариант B — **конкретный менеджер**

```env
SBIS_RECIPIENT_MODE=manager
SBIS_MANAGER_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
SBIS_AUTO_SUBMIT=true
```

См. также `scripts/sbis-manager-id.md`.

## Ошибка «необходимо назначить получателя»

Если `submit_appeal` / «На выполнение» (`Выполнить`) возвращает эту фразу при любом `SBIS_MANAGER_ID` и подразделении — **это настройка регламента в Saby**, не баг бота.

Что сделать админу Saby:

1. **Контакт-центр → Настройки → регламент «Обращение»** — на этапе **«Отправка»** указать **получателя / очередь** (подразделение или роль), куда можно передать обращение.
2. Учётке **`ai_agent` (ИИ Агент, id 27850)** — права на раздел **Обращения**, **обработка документов за других**, этап «Отправка» (см. [права обращений](https://saby.ru/help/another/helpdesk/law_appeal)).
3. Вручную создать одно обращение в UI, нажать **«На выполнение»**, выбрать получателя как для бота → выполнить `python scripts/sbis_show_recipient.py <guid>` и перенести **точные** `Подразделение` / `Сотрудник` в `.env`.

Пока submit не проходит: черновики создаются (`create_appeal` OK), менеджер открывает ссылку `open_url` из уведомления Telegram.

## Проверка

```powershell
python scripts\sbis_list_appeals.py
python workspace\scripts\sbis_api.py create_appeal '{"description":"Тест общей очереди","source":"test","channel_id":"test:queue-1"}'
```

В ответе: `draft: false` и `submit.submitted: true` — обращение должно быть в Контакт-центре.
