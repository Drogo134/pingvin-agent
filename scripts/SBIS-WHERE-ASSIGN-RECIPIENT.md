# Где в Saby назначить получателя «РПК ПИНГВИН, ООО»

Ответ на вопрос: *«Где мы должны назначить его получателем? Никакой очереди у нас нет»*.

## Важно: отдельного раздела «Очередь» может не быть

В переписке «очередь» — это **не отдельное меню**, а **список допустимых получателей** на этапе регламента документа **«Обращение»** (этап **«Отправка»**, кнопка **«На выполнение»**).

Пока этот список пустой или там нет подразделения/менеджеров, API бота отвечает:  
*«Для запуска документа в документооборот необходимо назначить получателя»*.

---

## Куда зайти администратору (пошагово)

Нужны права **«Администратор системы»** или полный доступ к регламентам.

### Шаг 1. Редактор регламента «Обращение»

Один из путей (зависит от версии интерфейса):

1. **Saby** → **Настройки** (шестерёнка) → **Документооборот** / **Регламенты** / **Операции документов**  
2. Или: **Управление бизнес-процессами** → список регламентов  
3. Найти регламент с названием **`Обращение`** (тот же, что при ручном «+ → Документ → Обращение» в Контакт-центре).

Документация Saby: [Выбрать исполнителя этапа](https://saby.ru/help/regulations/perfomer), [Добавить этап операции](https://saby.ru/help/regulations/create/etap).

### Шаг 2. Этап «Отправка»

1. Открыть регламент **«Обращение»** на редактирование.  
2. Вкладка **«Прохождение»** (цепочка этапов).  
3. Выбрать этап типа **«Отправка документа»** / с названием **«Отправка»** (на нём в карточке обращения кнопки **«На выполнение»** и **«Решено»**).

### Шаг 3. Назначить исполнителя / получателя

На этом этапе:

1. Нажать **«Исполнитель не определён»** или **«+»**.  
2. Выбрать тип **Подразделение** (не «ИИ Агент»).  
3. Указать **`РПК ПИНГВИН, ООО`**.

Так все сотрудники подразделения с правами на [обращения в Контакт-центре](https://saby.ru/help/another/helpdesk/law_appeal) смогут видеть и забирать обращения после отправки.

**Альтернатива:** вместо подразделения — конкретные люди (Ткачук, Федоров и т.д.) или **роль** (если заведена роль оператора).

### Шаг 4. Сохранить регламент

Сохранить изменения. Проверка:

1. Создать тестовое обращение вручную.  
2. **«На выполнение»** → в списке должен появиться **«РПК ПИНГВИН, ООО»** (или выбранные сотрудники).  
3. После отправки обращение видно в **Контакт-центр → Обращения → Список** (не только черновик).

## Что НЕ делать

| Неправильно | Почему |
|-------------|--------|
| Получатель = **ИИ Агент** | Только создаёт черновики по API (`ai_agent`), заявки людям не раздаёт |
| Искать меню **«Очередь»** | Обычно его нет; настройка в **регламенте → этап → исполнитель** |
| Только права CRM / сделки | Нужен участок **Контакт-центр / Обращение**, не CRM-сделки |
| `SBIS_DEPARTMENT_ID=4` в `.env` | ID из кадров часто **не совпадает** с ID в документообороте |

---

## Права для `ai_agent` (отдельно от получателя)

Учётка **ai_agent** — **автор** обращений, не получатель.

Админу выдать ([права на обращения](https://saby.ru/help/another/helpdesk/law_appeal)):

- Контакт-центр / Служба поддержки — **Просмотр и изменение**  
- Контакт-центр / Служба поддержки / **Обращение** — **Просмотр и изменение**  
- **Обработка документов за других сотрудников** — **Разрешено**

## После настройки (техпроверка)

```powershell
python scripts\sbis_try_all_recipients.py
```

Ожидание: `submitted=True` при варианте с подразделением «РПК ПИНГВИН, ООО».

В `.env` бота:

```env
SBIS_RECIPIENT_MODE=auto
SBIS_DEPARTMENT_NAME=РПК ПИНГВИН, ООО
SBIS_AUTO_SUBMIT=true
```

## Пока регламент не настроен

Заявки **не теряются**:

- менеджерам в **Telegram** приходит ссылка на черновик (уже работает);  
- в Saby черновик открывается по ссылке, менеджер вручную жмёт **«На выполнение»** и выбирает получателя.
