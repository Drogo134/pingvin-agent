# Saby (СБИС) — настройка обращений без ручного поиска в UI

## Важно: это не «CRM-сделки»

Агент создаёт **Обращения** в разделе **Контакт-центр**:
https://online.sbis.ru/page/support-service

Это **не** модуль CRM «Сделки». В интерфейсе ищите **Контакт-центр → Обращения**, не «CRM».

---

## «ИИ Агент» — что это

| Имя в Saby | Что это | Нужно в .env? |
|------------|---------|----------------|
| **ИИ Агент** / `ai_agent` | Технический логин API (создаёт черновики) | `SBIS_LOGIN` / `SBIS_PASSWORD` — уже есть |
| **РПК ПИНГВИН, ООО** | Подразделение компании | `SBIS_DEPARTMENT_NAME` |
| **Васьков Игорь** и др. | Реальные сотрудники | `SBIS_MANAGER_ID` (если заявки одному человеку) |

**ИИ Агент — не очередь для заявок.** Его нельзя указать как получателя.

---

## Шаг 1 — узнать данные автоматически (без UI)

В PowerShell из папки проекта:

```powershell
python scripts\sbis_discover.py
```

Скрипт через API Saby выведет:
- список подразделений;
- список сотрудников (кроме уволенных);
- готовые строки для `.env`.

Уже найдено для РПК ПинГвин:

```env
SBIS_RECIPIENT_MODE=auto
SBIS_DEPARTMENT_NAME=РПК ПИНГВИН, ООО
# SBIS_DEPARTMENT_ID=4   ← ID из «СписокПодразделений» часто НЕ работает в документообороте
SBIS_AUTO_SUBMIT=true
```

**Проверено API:** при `SBIS_DEPARTMENT_ID=4` Saby отвечает «Подразделение с идентификатором '4' не найдено» — это кадровый справочник, не очередь Контакт-центра.

## Шаг 2 — почему обращений не видно в списке

Цепочка в Saby:

1. `create_appeal` → черновик («Ожидается отправка») — **это уже работает** (№57–69 в системе).
2. `submit_appeal` → отправка в документооборот — **сейчас блокируется**.

Ошибка от Saby:

> «Для запуска документа в документооборот необходимо назначить получателя»

или

> «У вас нет прав…» / «Не удалось определить работающего сотрудника»

**Причина:** у учётки `ai_agent` **недостаточно прав** на этап «Отправка» регламента «Обращение». Это настраивает **администратор Saby**, не в интерфейсе «где-то найти название».

## Как устроено обращение (этапы в API)

На этапе **«Отправка»** в Saby две кнопки (видно в `ПрочитатьДокумент` → `Этап` → `Действие`):

| Кнопка в UI | API `Действие` | Что делает |
|-------------|----------------|------------|
| Передать коллеге / **На выполнение** | `Выполнить` | Нужен **получатель** (сотрудник). После успеха обращение в Контакт-центре «в работе». |
| **Решено** | `Решено` | Закрывает обращение (получатель не нужен). |

Статусы в списке: черновик = `Ожидается отправка` (код 0). После `Выполнить` — другой `Состояние.Описание`.

Команда API: `execute_appeal_action` с `action`: `"Выполнить"` или `"Решено"`, `manager_id`: `"27850"` (ИИ Агент) и др.

### Права (не путать CRM и Контакт-центр)

- [Права CRM (сделки, чаты)](https://saby.ru/help/crm/settings/law) — **не то**, что нужно для обращений.
- **[Права на обращения в Контакт-центре](https://saby.ru/help/another/helpdesk/law_appeal)** — **главная страница для админа**.

Участки для `ai_agent` (ИИ Агент, id `27850`):

| Участок | Нужный уровень |
|---------|----------------|
| `Контакт-центр/Служба поддержки` | Просмотр и изменение |
| `Контакт-центр/Служба поддержки/Обращение` | Просмотр и изменение |
| Доп. ограничение «Обработка документов за других сотрудников» | **Разрешено** (иначе нельзя назначить получателя через API) |

Плюс в **настройках регламента «Обращение»** — кто входит в очередь получателей (операторы). Без этого API отвечает: «необходимо назначить получателя», даже для id `27850`.

## Шаг 3 — что попросить администратора Saby

Передайте в IT / админу Saby:

1. Учётка API: **`ai_agent`** (логин из `.env`).
2. Нужны права на регламент **«Обращение»** / Контакт-центр:
   - создание документов (уже есть);
   - **выполнение действия «Отправка» / «Выполнить»** на этапе документооборота;
   - назначение получателя (подразделение **«РПК ПИНГВИН, ООО»** или конкретный менеджер).
3. В регламенте «Обращение» / Контакт-центре указать, **кто может быть получателем** (подразделение или сотрудники с ролью в очереди).
4. Для теста submit после прав: `SBIS_RECIPIENT_MODE=manager` и `SBIS_MANAGER_ID` — сотрудник с **ДоступВСистему=Да** (см. `sbis_discover.py`: сейчас Ткачук `223`, Федоров `3374`; **не** «ИИ Агент»).

Сотрудники без доступа в Saby (например Васьков `30`) дают ошибку «Не удалось определить работающего сотрудника».

После выдачи прав:

```powershell
python scripts\sbis_submit_drafts.py
.\scripts\start-local.ps1
```

## Обходной путь (работает уже сейчас)

Пока Saby не настроен, бот **не теряет заявки**:

| Канал | Статус |
|-------|--------|
| **Telegram → менеджеры** | `notify_managers.py` — уведомление на все ID из `MANAGER_TELEGRAM_CHAT_IDS` |
| **Черновики в Saby** | Создаются с номерами; открыть по ссылке из `python scripts\sbis_list_appeals.py` |
| **Диалог с клиентом** | Расчёт, сбор ТЗ, handoff — по ТЗ |

Менеджер может вручную открыть черновик по ссылке и нажать «Отправить» в Saby.
